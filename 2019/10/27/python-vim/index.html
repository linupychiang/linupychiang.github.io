VIM和Python - 天作之合

原文：https://realpython.com/vim-and-python-a-match-made-in-heaven/

注：原文中部分实操出现的问题已修复

关于这部分内容，有人非常推荐 Sublime Text 3 。

作为一名高级开发人员，我觉得我有责任告诉你需要的唯一真正 Python 开发工具是—— VIM 。

VIM 无处不在，速度快，而且永远不会崩溃。 它可以做任何事情！

不过，VIM的缺点是配置起来很麻烦，但是不用担心。

本篇文章将向你介绍如何设置一个功能强大的 VIM 环境，使其能够适应 Python 的日常开发。

如果要充分理解本文，你至少应该对如何使用 VIM 及其命令模式有一个基本的了解。如果你刚刚接触 VIM ，在继续之前，你应该花点时间先了解一些 VIM 的基础知识。具体请查看 http://vim-adventures.com/ 或 http://www.openvim.com/ 。

4. 开始配置一个 IDE

此处不可能列出所有 VIM 特性，但是让我们快速地看一下一些适合 Python 开发的功能强大的开箱即用特性。

抛弃鼠标

VIM 最重要的特性可能是它不需要鼠标( VIM 的图形变体除外)。

起初，这可能看起来像一个可怕的想法，但是在你投入时间之后 （并且确实需要时间来学习 VIM 快捷键 ），这将加快你的整体工作流程！

如果你使用 :sp 打开一个文件，你将垂直分割布局(在当前文件下面打开新文件)。

如果将键反向到 :vs ，就会得到一个水平分割(打开当前文件右侧的新文件)。

Buffers 缓冲区

虽然 VIM 可以使用选项卡，但许多用户更喜欢缓冲区和拆分。 你可以将缓冲区理解为最近打开的文件。 VIM 提供了对最近缓冲区的轻松访问。 只需键入：:b <缓冲区名称或编号> 即可切换到打开的缓冲区。 （此处也可以自动完成。）

你还可以使用 ls 列出所有缓冲区。

  • 提示4:

    在 ls 输出的末尾，VIM 将提示按回车键继续。你可以键入 b <buffer number> ，并在仍然显示列表的同时立即选择缓冲区。这样做的好处是：可以节省点击键盘的时间，而且不必记住缓冲区号。
